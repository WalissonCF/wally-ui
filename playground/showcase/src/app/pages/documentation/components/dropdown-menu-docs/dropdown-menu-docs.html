[ Component Architecture ]

Dropdown Menu follows a composable architecture inspired by Radix UI and shadcn/ui. Each part is a separate component that can be composed together for maximum flexibility. 

wally-dropdown-menu                  Container (provides service)
├── wally-dropdown-menu-trigger     Clickable element
└── wally-dropdown-menu-content     Menu container
    ├── wally-dropdown-menu-label   Section header
    ├── wally-dropdown-menu-separator  Divider
    ├── wally-dropdown-menu-group   Group wrapper
    ├── wally-dropdown-menu-item    Menu item
    └── wally-dropdown-menu-sub     Submenu container
        ├── wally-dropdown-menu-sub-trigger
        └── wally-dropdown-menu-sub-content

[ API Reference ]

> DropdownMenu Props

Property Type Default Description
triggerMode 'click' | 'hover' 'click' How the dropdown opens - on click or hover

> DropdownMenuContent Props

Property Type Default Description
position 'top' | 'bottom' | 'left' | 'right' 'bottom' Preferred position. Auto-adjusts if no space available

> DropdownMenuItem Props & Events

Property Type Default Description
disabled boolean false Disables the menu item
(click) OutputEmitterRef<void> - Emitted when item is clicked. Menu closes automatically

> DropdownMenuGroup Props

Property Type Default Description
ariaLabel string '' Accessible label for screen readers

[ Key Features ]

> Composable Architecture

Build complex menus by composing small, focused components. Full control over HTML structure and styling.

> Viewport-Aware Positioning

Automatically adjusts position to stay within viewport. Detects available space and chooses optimal placement.

> Click Outside to Close

Automatically closes when clicking outside. Uses efficient event delegation for performance.

> Hover Support with Smart Delays

200ms delay before opening prevents accidental triggers. 100-300ms close delay allows smooth mouse movement.

> Nested Submenus

Unlimited nesting depth. Each submenu has independent state management. Opens on hover with smooth transitions.

> Full Accessibility

ARIA roles (menu, menuitem, group). Proper focus management. Screen reader announcements. Semantic HTML structure.

> Smooth Animations

Tailwind CSS transitions for fade-in effects. 200ms duration with ease-out timing. GPU-accelerated transforms.
